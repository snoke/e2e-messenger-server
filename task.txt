# Refactor Task List (Updated)

Leitplanken (Kurzfassung):
- Thin Orchestrator: Runtime koordiniert nur, keine fachlichen Entscheidungsbloecke.
- State Access Budget: Runtime darf nicht breit/unstrukturiert auf viele State-Maps/Sets/Refs zugreifen.
- Inbound/Outbound strikt trennen.
- Refactor Completeness: Refactor ist unvollstaendig, wenn mehrere Fachbloecke oder grosse Closures bleiben.
- Soft Size Guardrail: Runtime 200–500 LOC Zielbereich (500–800 nur begruendet).
